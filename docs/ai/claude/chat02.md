"""
downloader에서 파일 형식이나 다운로드 방식(로직)이 다른 경우, 별도의 endpoint를 구성해야 할 것 같은데, text 형식 binary 형식, youtube 정도만 구분하면 될까요?
"""

  ● 설계 관점에서 먼저 정리하겠습니다.
  
  현재 코드의 실제 로직 분기
                                                                                                   
  요청 URL
   ├─ yt-dlp 처리   → youtube / video / m3u8 / HLS
   └─ requests 처리 → image / file / (text도 동일)

  text와 binary는 다운로드 로직이 동일합니다 (requests.get + stream). 저장 후 파일 내용이 텍스트냐
  바이너리냐의 차이만 있고, 코드 분기가 생기지 않습니다.

  따라서 실질적인 분기는 2가지입니다:

  ┌─────────┬─────────────────────┬───────────────────────────────┐
  │  구분   │      처리 방식      │             대상              │
  ├─────────┼─────────────────────┼───────────────────────────────┤
  │ direct  │ requests 스트리밍   │ 이미지, 파일, 텍스트, HTML 등 │
  ├─────────┼─────────────────────┼───────────────────────────────┤
  │ youtube │ yt-dlp 서브프로세스 │ YouTube, m3u8, HLS, DASH      │
  └─────────┴─────────────────────┴───────────────────────────────┘

  ---
  권장 구조

  POST /download          ← 자동 감지 (Chrome 확장에서 주로 사용)
  POST /download/direct   ← 직접 URL 다운로드 (명시적 호출용)
  POST /download/youtube  ← yt-dlp 다운로드 (quality/subtitles 등 옵션 풍부)

  - /download는 편의용 auto-dispatch로 유지
  - /download/youtube는 quality, audio_quality, subtitles 전용 스키마
  - /download/direct는 단순하게 url + directory + filename만

  ---
  이 구조로 리팩터링할까요?

===============

  ---
  공통 필드 (모든 엔드포인트)

  ┌─────────────┬──────┬────────────────────────────────────────────────────────┐
  │    필드     │ 타입 │                          설명                          │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ url         │ str  │ 다운로드 URL                                           │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ directory   │ str  │ 저장 경로                                              │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ filename    │ str? │ 저장 파일명 (없으면 자동)                              │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ overwrite   │ bool │ 동일 파일 덮어쓰기 여부                                │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ requester   │ str? │ 요청자 식별자 (로깅용)                                 │
  ├─────────────┼──────┼────────────────────────────────────────────────────────┤
  │ source_page │ str? │ 다운로드를 트리거한 페이지 URL (Referer 헤더 + 로깅용) │
  └─────────────┴──────┴────────────────────────────────────────────────────────┘

  ---
  /download/youtube 추가 필드

  ┌──────────────────────┬──────┬─────────────────────────────────────────────┐
  │         필드         │ 타입 │                    설명                     │
  ├──────────────────────┼──────┼─────────────────────────────────────────────┤
  │ quality              │ str  │ 해상도                                      │
  ├──────────────────────┼──────┼─────────────────────────────────────────────┤
  │ audio_quality        │ str  │ 오디오 품질                                 │
  ├──────────────────────┼──────┼─────────────────────────────────────────────┤
  │ subtitles            │ str? │ 자막 언어                                   │
  ├──────────────────────┼──────┼─────────────────────────────────────────────┤
  │ cookies_from_browser │ str? │ 로그인 필요 영상용 ("chrome", "firefox" 등) │
  ├──────────────────────┼──────┼─────────────────────────────────────────────┤
  │ playlist_items       │ str? │ 재생목록 범위 ("1-3", "1,3,5")              │
  └──────────────────────┴──────┴─────────────────────────────────────────────┘

  ---
  /download/direct 추가 필드

  ┌─────────┬───────┬─────────────────────────────────┐
  │  필드   │ 타입  │              설명               │
  ├─────────┼───────┼─────────────────────────────────┤
  │ headers │ dict? │ 커스텀 요청 헤더 (인증 토큰 등) │
  └─────────┴───────┴─────────────────────────────────┘

  ---
  사용자 정보에 대해

  이 서버는 로컬 단일 사용자 환경이므로, 별도 사용자 DB보다는 requester 문자열 + source_page 정도로
   로깅 목적만 충족하면 충분합니다. 실제 인증은 x-token 헤더로 이미 처리하고 있습니다.

  ---
  이 구조로 엔드포인트 3개(/download, /download/direct, /download/youtube)를 구성하고 각각 전용    
  스키마로 리팩터링할까요?

  리팩터링 요약

  schemas.py          ← Pydantic 모델 분리 (신규)
    ├─ BaseDownloadRequest      공통 필드
    ├─ DirectDownloadRequest    + headers
    ├─ YoutubeDownloadRequest   + quality / audio_quality / subtitles / cookies / playlist
    └─ AutoDownloadRequest      모든 필드 포함

  downloader.py
    ├─ download_direct()        requests 스트리밍
    ├─ download_youtube()       yt-dlp 서브프로세스
    └─ download_auto()          타입 감지 후 위 둘로 분기

  main.py
    ├─ POST /download           자동 감지
    ├─ POST /download/direct    직접 다운로드
    └─ POST /download/youtube   yt-dlp 다운로드
